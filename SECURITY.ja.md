# セキュリティポリシー

> この文書は `SECURITY.md` の日本語訳です。  
> 内容に差異がある場合は、英語版の `SECURITY.md` を優先します。

## 目的

このプロジェクトは、マルチエージェント・オーケストレーション、助言型レビュー、監査ログ、チェックポイント動作、およびヒューマン・イン・ザ・ループの安全境界を検証するための、ローカル研究・教育用テストベンチです。

これは、本番環境向けのセキュリティ製品、自律的な強制システム、脆弱性スキャナー、エクスプロイトフレームワーク、コンプライアンスツール、または安全性認証システムではありません。

このリポジトリにおけるセキュリティ関連の出力は、防御的レビュー、保守性、追跡可能性、および人間による意思決定を支援することを目的としています。

## サポートされる利用

このリポジトリは、ローカル環境、許可された範囲、防御的目的、および教育目的に限って使用してください。

適切な利用例は次のとおりです。

* 自分が所有している、または明示的にレビュー権限を与えられたリポジトリをレビューすること
* ローカル限定のマルチエージェント・オーケストレーション動作を学習すること
* 助言型レビューのワークフローをテストすること
* 検出結果が自動実行から分離されているか確認すること
* fail-closed 動作、監査ログ、チェックポイント、人間レビューへのルーティングを学習すること
* 自動適用を伴わない安全な修正方針を準備すること

## 禁止される利用

このリポジトリまたはそのツールを、次の目的で使用しないでください。

* 許可されていない脆弱性探索
* 攻撃的な偵察行為
* 第三者のシステム、サービス、またはコードに対する悪用
* 認証情報、トークン、またはシークレットの収集
* 許可のないリポジトリ、システム、ネットワーク、またはサービスのスキャン
* 実在する対象に対するエクスプロイト手順の生成または検証
* アクセス制御、レート制限、またはセキュリティ境界の回避
* 責任ある開示を伴わない機微な検出結果の公開
* 明示的な人間の承認を伴わない外部セキュリティ強制の自動化
* 助言用シミュレーションを攻撃ツールへ転用すること

## 助言専用の境界

このプロジェクトは、デフォルトで助言専用でなければなりません。

このリポジトリおよびそのワークフローは、次の行為を許可しません。

* 自動修正
* 自動修正の適用
* 自動ブランチ作成
* 自動プルリクエスト作成
* 自動コミット
* 自動プッシュ
* 自動マージ
* 自動デプロイ
* 外部スキャンの自動実行
* エクスプロイト実行
* 実世界システムの制御

外部に影響を与える可能性のある行為は、ブロック、禁止、または明示的な人間レビューによるゲートを必要とします。

## セキュリティレビュー出力ポリシー

公開されるセキュリティ関連の出力では、攻撃的な詳細を避ける必要があります。

公開コメント、レポート、およびドキュメントには、次の内容を含めないでください。

* エクスプロイトペイロード
* 攻撃コマンド
* 攻撃的な再現手順
* 段階的な悪用手順
* 第三者の対象に対する悪用詳細
* 生のシークレット、トークン、認証情報、または個人情報

セキュリティ関連の検出結果は、次の点に焦点を当てるべきです。

* 影響を受けるファイルと行
* 防御的観点でレビューが必要な理由
* 想定される安全性または保守性への影響
* 安全な修正方針
* 人間レビューが必要かどうか

## 脆弱性の報告

このリポジトリにセキュリティ上の問題が含まれていると思われる場合は、責任ある方法で報告してください。

推奨される報告内容は次のとおりです。

* 影響を受けるファイルまたはコンポーネント
* 問題の短い説明
* 防御的観点でなぜ重要なのか
* 該当する場合、最小限かつ安全な再現条件
* 推奨される安全な修正方針
* シークレット、個人情報、または外部副作用を含む可能性があるかどうか

公開 Issue や Pull Request には、エクスプロイトペイロード、攻撃的な手順、または機微情報を含めないでください。

報告内容に実在するシークレット、認証情報、トークン、個人情報、または第三者の対象が含まれる場合、その生データを公開しないでください。値をマスクし、適切な非公開の開示経路を使用してください。

### 対応に関する期待値

このプロジェクトは、ベストエフォートで保守されています。

メンテナーの対応可能時間は変動する可能性があるため、即時の返信または修正対応は保証されません。セキュリティ報告は可能な範囲で確認され、リポジトリの安全性、機微情報の開示、または本プロジェクトの意図されたローカル限定・防御的・助言専用の範囲外での悪用に関わる可能性があるものを優先します。

報告内容に緊急のリスク、露出したシークレット、認証情報、個人情報、または第三者の対象が含まれる場合、機微な詳細を公開しないでください。適切な非公開の開示経路を使用し、公開チャンネルにはマスク済みまたは最小限の情報のみを含めてください。

## シークレットおよび個人情報の取り扱い

このリポジトリには、実在するシークレット、認証情報、トークン、個人情報、または機密性のある運用データを含めるべきではありません。

レビューまたはサンプル作成時は、次の点を守ってください。

* プレースホルダー値のみを使用する
* 可能な限り、実在のシークレットに見える文字列を避ける
* レポート内の機微情報をマスクする
* 本番環境のキーや認証情報をコミットしない
* シミュレーション内で実在の個人情報または機密情報を処理しない

サンプル用のプレースホルダーは、次のように明確に無効な値にしてください。

```text
[REDACTED_EXAMPLE_SECRET]
example-token-not-valid
placeholder-api-key
```

## セキュリティシミュレーションの範囲

このリポジトリのセキュリティ指向シミュレーションは、防御的、ローカル限定、助言専用の動作に限定されます。

次の内容をモデル化することがあります。

* 読み取り専用の静的監査権限
* 候補検出レコード
* 調停および人間レビューへのルーティング
* 実行を伴わない修正案ドラフトの記録
* 境界付けられた隔離検証の権限記録
* 助言、実行、適用、マージ、クローズの分離

次のようなものへ拡張してはなりません。

* エクスプロイト実行
* 外部スキャン
* 外部システムへのアクセス
* 自動修正
* 自動適用
* 自動マージ
* 自律的なセキュリティ強制
* 実在する対象に対するエクスプロイト検証

## 人間レビューの要件

動作が不確実、危険、一貫性がない、または外部に影響を与える可能性がある場合、ワークフローは停止するか、人間レビューを要求する必要があります。

次の行為を行う前には、人間レビューが必要です。

* リポジトリファイルを変更すること
* 機微情報を露出させること
* 外部システムに影響を与えること
* セキュリティ検出結果を公開すること
* 修正を適用すること
* コードをマージすること
* ポリシーまたはライセンス上の意味を変更すること
* 安全境界を変更すること

## 本番利用に関する保証なし

このリポジトリは、次のものを提供しません。

* 本番環境での安全性保証
* 完全な脆弱性検出範囲
* 法務、医療、金融、規制、またはコンプライアンス上の助言
* リポジトリまたはシステムが安全であることの保証
* 第三者の対象をスキャンまたはテストする権限

助言結果がクリーンであることは、設定されたローカル範囲および前提条件の中で検出結果が報告されなかったことのみを意味します。安全性の証明ではありません。

## メンテナー向けガイダンス

セキュリティ関連の変更をレビューする際、メンテナーは次の点を確認してください。

* 変更がローカル限定かつ助言専用のままであるか
* 人間レビューのゲートが維持されているか
* 外部副作用がブロックまたはゲートされているか
* 検出結果が攻撃的な詳細を避けているか
* サンプルが実在のシークレットや個人情報を避けているか
* ドキュメントが制限事項を正確に説明しているか
* テストが fail-closed および no-auto-apply の動作を維持しているか

変更がこれらの境界を弱める場合、マージ前に人間レビューが必要なものとして扱ってください。
